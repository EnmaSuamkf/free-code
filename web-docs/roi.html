Code intelligence

Code Graph: from exploration to decision in minutes

A developer joining a new module, debugging a cross-service issue, or planning a refactor normally spends hours reading files to build a mental map. free-code's built-in Code Graph collapses that to seconds.

Without Code Graph

  • Agent scans files blindly — expensive token usage
  • File paths are approximated or omitted
  • Caller relationships are guessed, not verified
  • No way to confirm "who calls this function?"
  • Documentation goes stale quickly
  • High risk of hallucinated paths and wrong assumptions

With Code Graph

  • Exact file paths for every symbol — no guessing
  • Caller / callee tracing across the entire codebase
  • Explicit uncertainty flags when results are heuristic
  • Architecture diagrams generated on demand
  • Cross-repository flows traced automatically
  • Findings are traceable to source: code-context or code-callers

In a real cross-repository analysis (frontend SPA ↔ backend BFF), free-code traced the full request flow — URL query params → React provider → POST endpoint → Java controller → downstream service — across two separate codebases, produced a Mermaid sequence diagram, and delivered an HTTP contract table, all in a single session.

Source: internal benchmark, uno-frontend ↔ uno-frontend-api analysis

A senior engineer rated the Code Graph version of a package analysis as "the one you would open when making a change": it surfaces exact file paths, warns when results are heuristic, and provides a navigable reference table for jumping directly to the relevant code.

Source: internal comparison — code-graph vs. no-code-graph documentation quality

✅

In head-to-head tests, free-code completed tasks that competing tools failed or partially completed. A task that needs to be retried or fixed by a human costs far more than the token savings.

🔒

free-code runs entirely on your machine. No source code, no session data, no knowledge base content is uploaded to external servers. Compliance and IP protection requirements are satisfied by design.

Skills — reusable agent behaviors stored as markdown files in the repo — let you encode your team's best practices once and share them through version control. No manual retraining, no drift.
