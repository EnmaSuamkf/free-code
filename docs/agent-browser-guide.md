# Using the agent-controlled browser

This guide covers two ways to use the plugin's visual browser:

1. Opening a generic page from the `Browser` button.
2. Connecting the agent to a real Chrome instance via CDP to use Gemini while already logged in.

## Requirements

- `agent-browser` installed and available on `PATH`.
- `free-code.noExtensions` set to `false`.
- The `vscode-free-code` plugin installed and reloaded in Cursor/VS Code.
- For Gemini: the `gemini-browser` and `gemini-download` skills enabled in `/pick-skill`.

## Option 1: open a generic browser from the plugin

Use this option for pages that don't require complex login flows.

1. Open the **Free Code** panel.
2. Click `Browser`.
3. Enter the URL, for example:

```text
https://example.com
```

4. Optionally enter an initial goal, for example:

```text
Inspect the page and wait for my next instruction.
```

5. Click `Open visible browser`.

The plugin asks the agent to open a visible window with `agent_browser`, using a call equivalent to:

```json
{ "args": ["--headed", "open", "https://example.com/"], "sessionMode": "fresh" }
```

The agent should then take a snapshot:

```json
{ "args": ["snapshot", "-i"] }
```

From there you can issue natural-language actions:

```text
click the More information link
```

## Option 2: use real Chrome with CDP for Gemini

Use this option when Google blocks login with the message:

```text
Couldn't sign you in
This browser or app may not be secure.
```

The idea is to open a regular Chrome window with a debug port. You log in manually there, and then the agent connects to that Chrome instance.

### 1) Open Chrome with remote debugging

On macOS:

```bash
/Applications/Google\ Chrome.app/Contents/MacOS/Google\ Chrome \
  --remote-debugging-port=9222 \
  --user-data-dir="$HOME/.free-code/chrome-debug-profile"
```

This opens a new Chrome window using a persistent profile stored at:

```text
~/.free-code/chrome-debug-profile
```

### 2) Log in to Gemini

In that Chrome window:

1. Open `https://gemini.google.com/app`.
2. Log in manually.
3. Leave that window open.

Do not close it while the agent is using it.

### 3) Connect the agent to that Chrome

In the plugin chat, ask:

```text
open Gemini using agent_browser with --cdp http://127.0.0.1:9222
```

The expected call is:

```json
{
  "args": [
    "--cdp",
    "http://127.0.0.1:9222",
    "open",
    "https://gemini.google.com/app"
  ],
  "sessionMode": "fresh"
}
```

Afterwards the agent should take a snapshot:

```json
{ "args": ["snapshot", "-i"] }
```

## Gemini commands: ask, download, and open

Once connected to Gemini, use the command:

```text
/gemini ask search for information about total battle
```

This command loads the `gemini-browser` skill in the background and sends the text to Gemini.
If the session was already open before the command was added, run `/reload` once.

To download a file generated by Gemini:

```text
/gemini download <file_name>
```

This command uses the `gemini-download` skill to open the attachment in the viewer and execute the download with an absolute path in the project.

To open Gemini via CDP with an optional chat:

```text
/gemini open
```

```text
/gemini open <chat_id>
```

`/gemini open` executes:

```json
{
  "args": [
    "--cdp",
    "http://127.0.0.1:9222",
    "open",
    "https://gemini.google.com/app"
  ],
  "sessionMode": "fresh"
}
```

If you pass `<chat_id>`, it opens:

```json
{
  "args": [
    "--cdp",
    "http://127.0.0.1:9222",
    "open",
    "https://gemini.google.com/app/<chat_id>"
  ],
  "sessionMode": "fresh"
}
```

You can also do it manually:

```text
write the message "search for information about total battle" in Gemini
```

Then:

```text
send the message to Gemini
```

With the `gemini-browser` skill, the agent should:

1. Type the message into:

```text
div[aria-label="Enter a prompt for Gemini"]
```

2. Before sending, take `snapshot -i` and verify that input exists.
3. Click:

```text
[aria-label="Send message"]
```

4. Run `sleep 10` in bash in the foreground and wait for it to finish; do not use `agent_browser wait`.
5. Take another `snapshot -i` to check for a response. If there is none, repeat `sleep 10` and `snapshot -i` up to 5 times.

You can also phrase it as a single sentence:

```text
write the message "search for information about total battle" in Gemini and send it
```

## Common issues

### `agent_browser` does not appear

Check:

- `free-code.noExtensions` is set to `false`.
- `agent-browser` is on `PATH`.
- The plugin uses a recent `free-code`.

In the chat you can run:

```text
/tools
```

`agent_browser` should appear in the tool list.

### The skill is not being used

Open:

```text
/pick-skill
```

Enable `gemini-browser` and `gemini-download`.

To force it:

```text
use the gemini-browser skill to write the message "search for information about total battle" in Gemini and send it
```

### Cannot move/copy from `~/Downloads` (macOS)

If you want to manually move or copy something downloaded from `Downloads`, macOS may block access from the process running commands.

Enable permissions in:

- **System Settings → Privacy & Security → Files and Folders**
- (alternative) **Full Disk Access**

And grant `Downloads` access to Cursor/terminal.
If you want to avoid depending on that permission, use `/gemini download ...` to save directly to the project.

### Google keeps asking to log in again

Always launch Chrome with the same `--user-data-dir`:

```bash
--user-data-dir="$HOME/.free-code/chrome-debug-profile"
```

That profile stores your session.
